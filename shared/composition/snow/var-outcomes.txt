# Populated from a real `mkosi --profile snow build` (2026-07-14) the same
# way as shared/composition/cayo/var-outcomes.txt: the audit's own

# --- image-metadata: belongs to the immutable root -------------------------
# Same dpkg relocation as shared/composition/cayo/var-outcomes.txt: this
# build (bootc `snow`, unaffected by the native-gated relocation) shows the
# real, populated /var/lib/dpkg directory (~6100 individual files under
# lib/dpkg/info, lib/dpkg/alternatives, lib/dpkg/triggers, ...) — the exact
# opposite shape from a native build's single relocation-symlink candidate.
# One bare-`*` glob covers both shapes; see the cayo map's longer comment
# for why two separate globs would make the audit's stale-glob check
# spuriously fail depending on which output format is being built.

# FLAGGED GAP (not fixed by this task, see phase 2 task report): aspell's
# compiled dictionary binaries (*.rws) are NOT dpkg-tracked content — they
# are generated at package-configure time by the aspell-autobuildhash
# trigger (observed live in the build log) directly into /var/lib/aspell,
# separately from the static *.alias files that DO ship in /usr/lib/aspell.
